# 개인 기획 / 프로토타입 실험 기록

스위프 웹 14기 팀빌딩을 위해 정리한 개인 기획 및 프로토타입 실험 기록입니다.  
완성 서비스가 아니라, 아이디어와 사용자 흐름을 확인하기 위해 개인적으로 만든 테스트용 프로토타입입니다.

---

## 1. Speak Up

### 한 줄 소개
상황 기반 영어 말하기 연습 앱 프로토타입입니다.

### 문제 정의
영어 표현을 외워도 실제 상황에서 내 말로 바꿔 말하기 어려운 문제에 관심을 두었습니다.

### 핵심 흐름
상황 입력 → 대화 예시 확인 → 핵심 표현 학습 → 내 문장 만들기 → AI 피드백

### 해본 것
- 사용자 문제 정의
- 핵심 기능 정리
- 사용자 흐름 설계
- AI 도구를 활용한 프로토타입 제작
- Vercel 배포 테스트

### 프로토타입 링크
👉 https://english-speaking-app-blue.vercel.app/

---

## 2. NAA

### 한 줄 소개
계획을 세워도 실행으로 옮기기 어려운 사람을 위한 AI 실행 보조 앱 아이디어입니다.

### 문제 정의
할 일을 정해도 시작하지 못하거나, 계획이 복잡해질수록 실행이 막히는 문제에 관심을 두었습니다.

### 핵심 방향
전체 일정관리보다 “오늘 할 한 가지”를 정하고 바로 시작할 수 있도록 돕는 작은 실행 흐름에 집중했습니다.

### 해본 것
- 문제 정의
- 핵심 기능 정리
- 사용자 흐름 기획
- 미루기 진단 테스트 아이디어 정리

---

### 프로토타입 링크
👉 https://naa-app.vercel.app/

---

## 3. Ranking Party Game

### 한 줄 소개
친구들과 함께 즐길 수 있는 설문 예측 기반 파티 게임 웹앱 프로토타입입니다.

### 문제 정의
술자리나 모임에서 가볍게 즐길 수 있으면서도, 서로의 생각을 맞혀보는 방식의 참여형 게임을 만들고 싶었습니다.

### 핵심 흐름
질문 확인 → 각자 예상 순위 선택 → 실제 설문 결과 확인 → 예측과 실제 결과의 차이에 따라 점수/벌칙 적용

### 해본 것
- 게임 규칙 설계
- 질문/카테고리 흐름 정리
- 결과 비교 로직 기획
- 웹앱 프로토타입 제작
- Vercel 배포 테스트

### 프로토타입 링크
👉 https://ranking-party-game-plum.vercel.app

이번 스위프 프로젝트에서도 아이디어를 크게 벌리기보다, 팀원들과 함께 6주 안에 구현 가능한 핵심 경험을 먼저 정하고 완주하는 데 집중하고 싶습니다.

개발자로서 깊게 구현할 수 있는 수준은 아니지만, AI 도구와 VS Code/Vercel 등을 활용해 앱/웹 아이디어를 프로토타입 형태로 실험해본 경험이 있습니다.
